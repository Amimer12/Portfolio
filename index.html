Bank Al Salam — Internship Project
End-to-End MLOps Platform for Bank Budget Prediction

My biggest project to date, built during an internship with Bank Al Salam. It's a complete MLOps platform for budget prediction, covering the full lifecycle from raw data to a model running in production — not a notebook, an actual deployed system with a user interface on top.

The platform lets a non-technical user upload raw financial data, which is automatically validated and pushed through an ETL pipeline. From there, model training kicks off automatically, results get evaluated against tracked baselines, and approved models move into production — all visualized through interactive dashboards built for both technical and business users to monitor performance over time.

What it does
  • Interactive dashboards for visualizing model performance, predictions, and data drift
  • Raw data ingestion interface with automated ETL processing
  • Automated model training and evaluation, fully tracked and reproducible
  • Production deployment pipeline — from trained model to served endpoint
  • LightGBM as the core forecasting model, chosen for speed and tabular performance
  • Experiment tracking and model registry with MLflow
  • FastAPI backend serving predictions and orchestrating the pipeline
  • React frontend for the dashboards and data entry interface
  • Containerized with Docker, orchestrated with Kubernetes for scalable deployment

Streaming Big Data
Personal / Academic Project
Real-Time Taxi Data Pipeline — Kafka, MongoDB & Cassandra

A real-time Big Data pipeline that streams and processes data from New York City's taxi service. The goal was twofold: build a working streaming architecture with Kafka, and directly compare two distributed NoSQL databases — MongoDB and Cassandra — as consumers of the same stream.

What it does
  • Kafka brokers handle the streaming layer, with a dedicated producer replaying NYC taxi trip data
  • Two parallel consumers write the same stream into MongoDB (sharded cluster) and Cassandra (multiple replication factors), so throughput and query behavior can be compared directly
  • Fully containerized with Docker Compose, with separate compose stacks for brokers, Mongo shards, and Cassandra nodes
  • Includes a web app layer for inspecting and visualizing the ingested data

Kafka MongoDB Cassandra Docker Compose Python

A geospatial data engineering project analyzing the spatial distribution of urban waste collection points.
